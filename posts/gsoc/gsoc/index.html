Introduction to SBOL Link to heading

My prior experience with the SBOL data format was visiting the SynBioHub website.
I understood it as a valuable location to hold biological data, but I didn’t imagine I would start working with this data format until the program began.

The aim of this project was to create a pipeline that could transform data represented in SBOL format into preprocessed data formats (vectors) usable for machine learning training.

  • SynBioHub is the largest database of standardized biological designs encoded in SBOL.
  • It is a great place to obtain large amounts of training data for ML models.
  • Classical ML models can be trained on:
    • RNA expression values
    • Encoded DNA sequences
  • These can be represented in SBOL and uploaded to SynBioHub.

However, because SBOL is encoded in a triple-store XML format, these values cannot be directly passed into ML models (which expect numeric features).
Thus, researchers face the challenge of encoding SBOL data into ML-friendly formats, which is infeasible without ML experience.

As a result, SynBioHub currently lacks accessible, preprocessed datasets, limiting the ability of researchers to efficiently develop ML models.

Task One — Encoding Data into SBOL Link to heading

My first task was to encode data from a research paper into SBOL format.

  • The researchers were studying promoter sequences in the E. coli genome.
  • They uploaded their experimental data to GitHub.
  • They trained ML models to:
    • Predict whether a promoter sequence was active/inactive
    • Predict the expression level of the promoter

Since our project was about building an SBOL-to-ML pipeline, I used their dataset to test our package.

Tools and References Link to heading

  • SBOL specification → describes structure, logic, and reasoning of SBOL
  • pySBOL2 (Python package) → provides functions to generate SBOL files easily

Process Link to heading

  1. Read through the SBOL documentation.
  2. Decided on the best way to represent experimental rows in SBOL.
  3. Wrote code to loop through rows and encode them into SBOL files.
  4. Iteratively refined the representation.

At first, I created one giant SBOL file encoding ~17,000 rows.
Later, we decided to encode each row as an individual SBOL file and group them in a SynBioHub collection.

We also struggled with encoding the RNA expression level (our target variable).

  • Initially tried using SBOL attachments
  • Eventually switched to the SBOL Measurement class → allowed us to pull data via SPARQL queries

Results Link to heading

By the end of this process:

  • I generated SBOL files representing the research dataset.
  • Encoding all rows was slow (took about a day).
  • Next step: upload the data to SynBioHub.

Luckily, SynBioHub has a well-documented API.

  • Used it to upload the SBOL files into a collection.
  • Verified we could pull data back via the API (simulating a package user workflow).

This confirmed our SBOL → SynBioHub → ML pipeline works end-to-end.
